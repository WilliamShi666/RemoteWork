Core LangGraph Concepts

LangGraph extends LangChain with powerful graph-based orchestration capabilities. Here are the fundamental building blocks you need to understand.

📊

StateGraph

The foundation of LangGraph. StateGraph manages the flow of data through your agent, maintaining state across multiple processing steps. It's like a state machine that orchestrates your AI workflow.

🔧

State

A TypedDict that holds all the data your agent needs. State persists across nodes and can be updated at each step. Think of it as the agent's working memory throughout execution.

📦

Nodes

Processing units that perform specific tasks. Each node is a function that takes the current state, performs operations, and returns an updated state. Nodes are where your business logic lives.

🔗

Edges

Connections between nodes that define execution flow. Edges can be simple (always go to next node) or conditional (choose next node based on state). They create your agent's decision tree.

🎯

Entry & Exit Points

Define where your graph starts and ends. Entry point is the first node to execute, while END marks completion. Multiple paths can lead to END based on your logic.

⚙️

Compilation

Transform your graph definition into an executable agent. The compile() method validates your graph structure and creates an optimized runtime that can process inputs efficiently.

Python - Basic LangGraph Structure 
# Define your state structure
class AgentState(TypedDict):
    input: str
    output: Optional[str]
    intermediate_data: Optional[Dict]

# Create the graph
workflow = StateGraph(AgentState)

# Add nodes (processing functions)
workflow.add_node("process", process_function)
workflow.add_node("analyze", analyze_function)

# Set entry point
workflow.set_entry_point("process")

# Add edges (flow control)
workflow.add_edge("process", "analyze")
workflow.add_edge("analyze", END)

# Compile and run
agent = workflow.compile()
result = agent.invoke({"input": "Hello LangGraph!"})

Three Powerful Agent Architectures

Learn how to build different types of agents for various use cases. Each architecture has its strengths and ideal applications.

Reactive Architecture

Fund Q&A Assistant

Fast, tool-based responses for immediate queries. Perfect for customer service, FAQ systems, and real-time information retrieval.

  • Immediate response time
  • Tool integration (search, calculations)
  • Stateless processing
  • Perfect for simple queries
Deliberative Architecture

Investment Research Analyst

Multi-stage reasoning for complex analysis. Ideal for research, planning, and tasks requiring deep understanding.

  • 5-stage processing pipeline
  • Maintains internal world model
  • Generates multiple hypotheses
  • Produces comprehensive reports
Hybrid Architecture

Wealth Management Advisor

Dynamically switches between fast and deep processing. Best for applications needing both quick responses and complex analysis.

  • Intelligent mode selection
  • Combines reactive & deliberative
  • Context-aware processing
  • Optimal resource utilization

This agent follows a 5-stage pipeline for comprehensive analysis and report generation.

# Define the state for deliberative processing
class ResearchAgentState(TypedDict):
    research_topic: str
    perception_data: Optional[Dict]
    world_model: Optional[Dict]
    reasoning_plans: Optional[List[Dict]]
    selected_plan: Optional[Dict]
    final_report: Optional[str]
    current_phase: str

# Create the workflow with 5 stages
workflow = StateGraph(ResearchAgentState)

# Add all processing nodes
workflow.add_node("perception", perception)      # Collect data
workflow.add_node("modeling", modeling)          # Build model
workflow.add_node("reasoning", reasoning)        # Generate plans
workflow.add_node("decision", decision)          # Select best

workflow.add_edge("reasoning", "decision")
workflow.add_edge("decision", "report")

Building a Hybrid Agent: Wealth Management Advisor

This agent intelligently chooses between reactive and deliberative modes based on query complexity.

Python - Hybrid Agent Dynamic Routing 
# Hybrid agent state includes processing mode
class WealthAdvisorState(TypedDict):
    user_query: str
    query_type: Optional[Literal["emergency", "analytical"]]
    processing_mode: Optional[Literal["reactive", "deliberative"]]
    final_response: Optional[str]

# Assessment function determines processing mode
def assess_query(state: WealthAdvisorState) -> WealthAdvisorState:
    # Use LLM to classify query complexity
    assessment = llm.invoke(f"Classify query: {state['user_query']}")
    
    if "simple" in assessment or "quick" in assessment:
        state["processing_mode"] = "reactive"
    else:
        state["processing_mode"] = "deliberative"
    
    return state

# Create hybrid workflow
workflow = StateGraph(WealthAdvisorState)

# Add nodes
workflow.add_node("assess", assess_query)
workflow.add_node("reactive", reactive_processing)
workflow.add_node("deliberative", deliberative_processing)
workflow.add_node("respond", generate_response)

# Dynamic routing based on assessment
workflow.add_conditional_edges(
    "assess",
    lambda x: x["processing_mode"],  # Route based on mode
    {
        "reactive": "reactive",
        "deliberative": "deliberative"
    }
)

# Both paths converge to response
workflow.add_edge("reactive", "respond")
workflow.add_edge("deliberative", "respond")
workflow.add_edge("respond", END)

Choosing the Right Agent Architecture

Compare the three architectures to select the best fit for your use case.

Feature Reactive Agent Deliberative Agent Hybrid Agent
Response Time Immediate (< 1s) Slower (5-30s) Variable (1-30s)
Complexity Handling Simple queries only Complex analysis Both simple and complex
State Management Minimal/Stateless Rich state across stages Adaptive state management
Resource Usage Low High Optimized per query
Best Use Cases Customer service, FAQs, Quick lookups Research, Planning, Analysis Multi-purpose assistants
Implementation Complexity Simple Complex Most Complex

When to Use Reactive

Choose reactive agents when you need immediate responses and have well-defined tools. Perfect for customer-facing applications where speed is critical and queries are straightforward.

When to Use Deliberative

Opt for deliberative agents when accuracy and depth matter more than speed. Ideal for research tasks, strategic planning, and scenarios requiring multi-step reasoning.

When to Use Hybrid

Select hybrid agents when your application faces diverse query types. Best for comprehensive AI assistants that need to balance responsiveness with analytical capabilities.

Get Started with LangGraph

Ready to build your own LangGraph agent? Follow these steps to create your first intelligent agent.

Install Dependencies

Set up your Python environment with LangGraph and required packages.

Bash 
pip install langgraph langchain langchain-community

Define Your State

Create a TypedDict that represents your agent's state structure.

Python 
from typing import TypedDict, Optional

class MyAgentState(TypedDict):
    user_input: str
    processed_data: Optional[dict]
    final_output: Optional[str]

Create Processing Nodes

Write functions that process your state and return updated versions.

Python 
def process_input(state: MyAgentState) -> MyAgentState:
    # Your processing logic here
    processed = {"result": f"Processed: {state['user_input']}"}
    return {**state, "processed_data": processed}

Build Your Graph

Connect nodes with edges to create your agent's workflow.

Python 
from langgraph.graph import StateGraph, END

workflow = StateGraph(MyAgentState)
workflow.add_node("process", process_input)
workflow.add_node("output", generate_output)

workflow.set_entry_point("process")
workflow.add_edge("process", "output")
workflow.add_edge("output", END)

agent = workflow.compile()

Run Your Agent

Invoke your compiled agent with initial state to get results.

Python 
result = agent.invoke({
    "user_input": "Hello, LangGraph!",
    "processed_data": None,
    "final_output": None
})

print(result["final_output"])
